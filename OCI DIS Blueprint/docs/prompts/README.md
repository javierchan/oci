# Historical Execution Prompts

The files in this directory are retained as implementation history. They may
refer to repository layouts, service counts, fixtures, or acceptance checks
that were valid when each prompt was written.

They are not active engineering contracts. Current work must follow, in order:

1. `AGENTS.md`
2. `README.md`
3. `.github/workflows/oci-dis-blueprint-quality.yml` at the repository root
4. current documents under `docs/architecture/`
5. `docs/reports/status-report.md`

Do not use a historical prompt to override current code, migrations, Compose,
CI, or validation evidence.
